Implementation: Outlier Detection¶

Detecting outliers in the data is extremely important in the data preprocessing step of any analysis. The presence of outliers can often skew results which take into consideration these data points. There are many "rules of thumb" for what constitutes an outlier in a dataset. Here, we will use Tukey's Method for identfying outliers: An outlier step is calculated as 1.5 times the interquartile range (IQR). A data point with a feature that is beyond an outlier step outside of the IQR for that feature is considered abnormal.

In the code block below, you will need to implement the following:

  • Assign the value of the 25th percentile for the given feature to Q1. Use np.percentile for this.
  • Assign the value of the 75th percentile for the given feature to Q3. Again, use np.percentile.
  • Assign the calculation of an outlier step for the given feature to step.
  • Optionally remove data points from the dataset by adding indices to the outliers list.

NOTE: If you choose to remove any outliers, ensure that the sample data does not contain any of these points!
Once you have performed this implementation, the dataset will be stored in the variable good_data.

Question 4¶

Are there any data points considered outliers for more than one feature based on the definition above? Should these data points be removed from the dataset? If any data points were added to the outliers list to be removed, explain why.

Answer:

Generally I would caution against removing outliers, so long as they are legitimate observations.

When building a statistical model, it is often handy to remove outliers which cause undue influence when simulating the 'average case'. This is because such observations, while often valid, do not make up the vast bulk of what we are trying to simulate and predict, and attempting to include such rare cases in the predictive model may serve to simply lower the model's overall accuracy - is it better to predict 90% of cases extremely well and make no attempt on the remaining 10% of cases, or predict 100% somewhat poorly. Removing outliers moves our goals towards the former and away from the latter. Additionally, outliers may make a model prone to overfitting.

I have identified five outlier samples which appear as outliers in more than one feature type. They are samples 65, 66, 75, 128, and 154. These samples specifically have been selected for exclusion, as being outliers for more than two features may indicate these are truly 'one off' observations - the likelihood of a single case appearing as a outlier for two features is so small that it is not a useful thing include in our model.

Question 6¶

What are the advantages to using a K-Means clustering algorithm? What are the advantages to using a Gaussian Mixture Model clustering algorithm? Given your observations about the wholesale customer data so far, which of the two algorithms will you use and why?

Answer: K Means looks to assign data points to one of K clusters - the value of K is assigned beforehand. Gaussian Mixture Model performs soft clustering, 'associating' data points with subpopulations with a percentage liklihood of the data point belonging to that cluster. K means classifies based on distance, wheras GMM clusters on a weighted distance - weighted according to variance. The advantages of Kmeans include the fact it is computationally efficient, and classifies data that falls into 'globs' well (that is, data that is distributed in such a way that it has natural, well defined clusters). Disadvantages of KMeans include if the clusters are uneven in size, the algorithm will not classify them correctly. Additionally, if the clusters are not 'globular', KMeans will not successfully select the 'intuitive' classification. The mixture model will tend to do much better with distributions of uneven size, but will be more computationally expensive. I would recommend using the GMM model for the customer data, as there are likely to be small 'islands' of various sizes in the dataset, which GMM can capture more easily.

Question 10¶

Companies will often run A/B tests when making small changes to their products or services to determine whether making that change will affect its customers positively or negatively. The wholesale distributor is considering changing its delivery service from currently 5 days a week to 3 days a week. However, the distributor will only make this change in delivery service for customers that react positively. How can the wholesale distributor use the customer segments to determine which customers, if any, would react positively to the change in delivery service?
Hint: Can we assume the change affects all customers equally? How can we determine which group of customers it affects the most?

Answer:

We cannot assume the change affects all customers equally.

An A/B approach would involve building four groups - taking two sets of samples from the two clusters. One of the Cluster 0 sample sets would recieve the new delivery service, while the other would retain the existing delivery service (control). The same setup is repeated with the Cluster 1 sample sets - one recieving the new service, and the other retaining the existing one. After some time has elapsed, data is captured from each group - either a survey of customer satisfaction, or taking a measurement of changed sales figures. We then can compare a cluster's "new service" group feedback to that cluster's "old service" group feedback. If the "new service" group gave more positive feedback, or saw an uptick in sales when compared to the "old service" group in the same cluster, we can assume that reaction to the new service is positive. Likewise for negative or neutral feedback. This process is repeated for the other cluster, and we get a view on 1) Whether cluster 0 customers will react positively to the change, and 2) whether cluster 1 customers will react positively to the change. From this the distributor may make the call as to whether to implement the new delivery service, and which customers will react in which ways to the change. The comparing the magnitude in difference between the "new service" and "old service" groups betwen the two clusters will tell the distributor which customer type would be most affected by this change.
